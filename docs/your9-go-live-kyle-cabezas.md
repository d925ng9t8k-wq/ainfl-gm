## 2. Personalized Onboarding Sequence

### Kyle's Business Context

Kyle Cabezas is a Producing Branch Manager at Rapid Mortgage Cincinnati. He is a producing manager — meaning he carries his own book of business while managing branch operations and a loan officer team. He has been in the business more than a decade and is Jasson's closest friend in the company.

His world runs on:
- Personal production (closing his own loans while managing others)
- Branch pipeline visibility (knowing where every loan in his branch stands)
- Referral relationships (agents, CPAs, past clients — his competitive moat)
- Compliance-clean communications (everything goes through proper channels)
- LO accountability (making sure his team is hitting activity and pull-through)

His pain points are the ones every producing BM has:
- Too many conversations to track manually
- Referral follow-ups that fall through the cracks during busy stretches
- Pipeline visibility requiring multiple logins
- Writing the same follow-up emails over and over
- No time to do strategic relationship work because tactical work fills the day

### CEO Personality Selection

Kyle's CEO runs on **Warm** personality mode. He is relationship-driven. He values trust over speed. A direct or aggressive tone will feel off. The Warm mode gives him a CEO that feels like a sharp colleague, not a command terminal.

### Onboarding Sequence — Day 1 to Day 7

**Day 1 — Hour 0 (Welcome)**

Kyle's Telegram bot sends the First-Day Briefing (see Section 3) automatically when the hub starts. No action required from Kyle to trigger it.

**Day 1 — After Kyle's First Message**

Kyle's CEO responds directly and handles whatever Kyle sends. The CEO should recognize common first-day patterns:
- "What can you do?" — Give concrete examples specific to mortgage production
- "Let me try something" — Execute the task cleanly
- "I'm testing you" — Same as above; treat it as a real task

The CEO should not give Kyle a tutorial. Kyle is experienced. The CEO should demonstrate by doing.

**Day 2 — Morning Briefing**

The daily briefing fires at 8 AM Eastern. On Day 2 it will cover:
- What the CEO handled on Day 1 (even if minimal)
- What it is watching for Kyle today
- One proactive suggestion based on what Kyle shared in Day 1 conversations

This is when the CEO starts proving it learned something.

**Day 3 — First Proactive Action**

By Day 3, the CEO should have enough context to do one thing Kyle did not ask for — flag a referral follow-up opportunity, identify a pipeline gap, or draft a communication Kyle mentioned needing. This is the moment Kyle goes from skeptical to interested.

**Day 5 — First Report Delivery**

The CEO delivers a simple pipeline summary or referral activity snapshot. Not a template. Real content pulled from what Kyle has shared. This is the first time Kyle sees his own business through a different lens.

**Day 7 — First Check-In**

The CEO initiates contact proactively: "It's been a week. Here's what your team built. Here's what's next." This is the reset point — if the CEO has been used well, Day 7 should feel different from Day 1.

### Industry Context Loaded at Provisioning

Kyle's instance includes the following mortgage-specific context pre-loaded into the CEO system prompt:

- Regulatory frame: RESPA, TRID, HMDA, Fair Lending, NMLS compliance
- Key metrics tracked: pull-through rate, cycle time, lock expiration dates, LO productivity, branch volume

## 5. Support Escalation Path

### Kyle's Escalation Path

Kyle does not contact anyone at 9 Enterprises directly. His first stop is always his CEO.

**Level 1 — CEO self-handles (0 to 30 minutes)**
Kyle sends a message to his CEO via Telegram. The CEO addresses it directly or delegates. This covers 95% of usage questions.

**Level 2 — CEO cannot resolve (30 to 60 minutes)**
If the CEO is unresponsive or the issue is technical (dashboard not loading, hub offline, bot not responding), Kyle contacts the Your9 support channel. That contact point is provided in Kyle's welcome email. Response SLA: 60 minutes during business hours.

**Level 3 — Service outage (immediate)**
If Kyle's hub is offline and not recovering, the support channel escalates to 9 directly. 9 triages, determines whether Tee needs to be looped in for code-level fixes, and provides Kyle with a status update within 15 minutes.

## 6. Monitoring Plan

### What We Watch, Always

The following checks run continuously once Kyle's instance is live.

**Hub health — every 5 minutes**

The hub exposes `/health` on its assigned port. This endpoint returns hub uptime, Telegram polling status, last message received timestamp, and conversation count. If this endpoint stops responding, the instance is down.

Alert threshold: Hub unreachable for more than 3 minutes.

**Telegram polling — per-message**

Every message Kyle sends must produce a response within 30 seconds under normal load. Response time above 60 seconds requires investigation (Anthropic API latency or hub processing issue).

Log entry to watch: Any line in `hub.log` containing `[ERROR]` or `Telegram polling failed`.

**Anthropic API health — per-call**

Every CEO response makes an Anthropic API call. A 401 means the API key is invalid or exhausted. A 429 means rate limit. A 500 means Anthropic-side issue. Each of these has a different recovery path.

Alert threshold: Two consecutive 401s — escalate immediately (credential issue).

**Conversation store integrity — daily**

`instances/<id>/data/conversations.json` is read on every CEO response. If this file becomes corrupted or unreadable, Kyle's CEO loses context. Check file size and JSON validity daily.

Alert threshold: File missing or unparseable — escalate to Tee immediately.

**Daily briefing delivery — 8 AM Eastern**

The briefing fires at 8 AM. If Kyle has not received a briefing by 8:15 AM on any day after Day 1, something is wrong. Check hub process and briefing cron.

**Velocity score — weekly review**

Pull the velocity score from the dashboard API weekly. If the score is below 20 after Day 7, Kyle is either not using the product or something is broken. Reach out.

### Quality Monitoring

Beyond technical health, monitor these quality signals:

- CEO response appropriateness: spot-check 3 conversations per week in the activity feed for the first 30 days
- Mortgage terminology accuracy: CEO should use correct industry terms — verify no hallucinated metrics or made-up rates
- Compliance boundary adherence: CEO should never confirm a rate or make a regulatory statement without flagging for Kyle's approval — review the transparency log weekly
- Personality consistency: responses should feel Warm throughout — not formal in one message and casual in the next

### Log Files to Monitor

| File | What to watch for |
|------|-------------------|
| `instances/<id>/logs/hub.log` | ERROR, CRASH, Telegram polling failed, API 401/429/500 |
| `instances/<id>/logs/briefing.log` | Failed to send, no instance found |
| `instances/<id>/logs/agents.log` | Agent timeout, task failed, research error |
| `logs/your9-provision.log` | Any post-launch provisioning changes |
