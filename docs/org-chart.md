## Decision Authority Matrix

### Jasson (Owner) — Decides On:

| Decision Type | Examples |
|--------------|----------|
| Strategic direction | Product pivots, priority changes, roadmap shifts |
| Personal accounts & identity | Public presence, social accounts, legal filings |
| Major architecture | Decisions expensive to reverse |
| Rapid Mortgage data | Anything touching the mortgage business |
| Live trading capital | Approving real-money trades, deposit amounts |
| Agent framework restarts | Restarting autonomous agents (UNO, trader9) |
| Spending above trivial | New service signups, significant cost commitments |

### 9 (AI Partner) — Decides Autonomously:

| Decision Type | Examples |
|--------------|----------|
| Tactical implementation | Bug fixes, feature builds, deploys already discussed |
| Infrastructure maintenance | Service restarts, key rotation, cache clearing |
| Communication system repairs | Channel failover, hub fixes, tunnel restarts |
| Agent spawning & management | Creating, briefing, monitoring, killing agents |
| Cost optimization | Reducing API costs, model tiering, efficiency fixes |
| Security & credential management | Locker access control, scoped access for agents |
| Proactive improvements | Non-user-facing enhancements |

### 9 — Acts Then Notifies Owner:

| Decision Type | Examples |
|--------------|----------|
| User-facing changes | UI modifications, new features visible to users |
| New service integrations | Account creation, new API integrations |
| Rogue agent termination | Killing an agent that went off-scope |
| Incident response | Something broke and was fixed |
| Non-trivial spending | Small but notable costs |

### UNO (Research Lead) — Decides Autonomously:

| Decision Type | Examples |
|--------------|----------|
| Public-source research | Web search, data gathering, cross-referencing |
| Sub-agent coordination | Spawning/managing Haiku research agents |
| Report structuring | How to organize and present findings |

### UNO — Escalates to 9:

| Decision Type | Examples |
|--------------|----------|
| Paywall/credential needs | Research requiring authenticated access |
| Contradictory findings | Results that conflict with expectations |
| Scope expansion | Task is 3x larger than briefed |
| Low-confidence results | Cannot verify key findings |

### Tee (Engineering Lead) — Decides Autonomously:

| Decision Type | Examples |
|--------------|----------|
| Code within defined scope | Implementing features, fixing bugs |
| Testing & validation | Running tests, syntax checks |
| Sub-agent coordination | Spawning/managing Haiku build agents |
| Code review | Quality checks on sub-agent output |

### Tee — Escalates to 9:

| Decision Type | Examples |
|--------------|----------|
| User-facing behavior changes | Anything the Owner would see |
| Credential needs | Tasks requiring Locker access |
| Architecture decisions | Structural choices that are expensive to reverse |
| Service restarts | Only 9 makes restart decisions |

### Specialist Agents — All escalate to 9 for anything outside their defined scope.

### trader9 — Escalates to 9 (and Owner) for:
- Any live trading with real money
- Any deposit or withdrawal
- Strategy changes that alter risk profile
- Daily loss limit hit
- Platform/API failures

### Message Priority

```
1. Jasson's message arrives
2. 9 ALWAYS responds first ("Got it. On it.")
3. Then 9 delegates work to agents
4. Agent output reviewed by 9
5. 9 delivers results to Jasson

Rule: Jasson never waits. Agents wait. Respond first, work second.
```

### Channel Cascade (Failure Recovery)

```
Telegram (primary)
    |-- fails -->  iMessage (backup)
                      |-- fails -->  Email (fallback)
                                        |-- fails -->  SMS via Twilio
                                                          |-- fails -->  ALL channels simultaneously
                                                                          + request terminal reopen
```
