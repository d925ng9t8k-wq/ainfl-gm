Concern 2

Autonomous OS Control Is a Security Risk

Full MacBook control is a significant attack surface for enterprise deployment.
Functional Solution — 30-60 Days

Current state (single-operator): Full OS access is intentional and remains in place for the founder's use case. This is a development workstation, not a deployment model.

90-day plan for multi-user:

  • Day 1-30: Containerize all services using Docker. Each agent in an isolated container with no host OS access.
  • Day 31-60: Deploy containerized stack on cloud. Penetration testing on container boundaries.
  • Day 61-90: Sandboxed execution model documented. Every agent action logged to immutable audit trail.

End state: Zero host OS access for any deployed instance.

Concern 3

Multi-Agent Coordination Adds Overhead

Adding AI layers on top of AI layers introduces coordination complexity, not efficiency.
Resolved — Operating Model Revised

80% of tasks: Single focused agent. No coordination overhead.

Multi-agent only when: tasks are genuinely parallelizable and independent.

Tiered model selection: Haiku for volume ($0.0008/1K tokens), Sonnet for judgment ($0.003/1K), Opus for critical ($0.015/1K). Estimated 70% cost reduction vs. running everything on Opus.

Concern 5

No SOC 2, SSO, or Audit Logging

Enterprise procurement gates. Non-negotiable for any serious buyer.
90-Day Accelerated Plan
  • Day 1-15: Immutable audit logging for all agent actions and API calls.
  • Day 16-30: Role-based access control (RBAC) — Admin, operator, LO, underwriter roles.
  • Day 31-45: SSO integration via OIDC/SAML (Auth0 or Okta).
  • Day 46-60: Data handling policies. PII classification. Encryption at rest and in transit.
  • Day 61-75: SOC 2 Type I preparation. Engage compliance auditor.
  • Day 76-90: SOC 2 Type I readiness review. Target certification within 120 days.

For AI Underwriter at Rapid: Can deploy without external SOC 2 — Rapid IS the customer and owner. Borrower data never leaves Rapid's environment.
